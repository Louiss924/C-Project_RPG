## Context

目前戰鬥介面中玩家的手牌是由隨機抽卡得到的順序，未經排序且邊框統一為白色（選取時為黃色）。為了提升戰鬥介面的視覺清晰度，需要對手牌進行依種類排序、加上種類文字，並將不同種類的卡牌邊框色彩化。

## Goals / Non-Goals

**Goals:**
- 在玩家手牌發生異動（抽卡、出卡、載入存檔）後，自動將手牌依據「攻擊、防禦、輔助」順序排序。
- 在戰鬥 UI 渲染卡牌時，加入種類標籤（例如 `[⚔️ 攻擊]`、`[🛡️ 防禦]`、`[🌳 輔助]`）。
- 將未選取卡牌的邊框依據種類分別上色（攻擊：淡紅、防禦：淡藍、輔助：淡綠），選取時維持黃色。

**Non-Goals:**
- 改變卡牌原本的功能、消耗或數值。
- 改變出牌的判定邏輯或方向鍵導航邏輯。

## Decisions

### 1. 手牌自動排序時機與實作
- **決策**：在 `Player` 類別中新增一個私有方法 `void sortHand()`。每當手牌被修改時（如 `drawCards()` 抽卡後、`playCard()` 出卡後、`setDeckState()` 載入狀態後），立即呼叫 `sortHand()` 進行排序。
- **排序規則**：
  1. 優先權 1：攻擊型（`DAMAGE`、`TRUE_DAMAGE`、`BREAK_ARMOR_DAMAGE`、`MULTI_DAMAGE`）
  2. 優先權 2：防禦型（`DEFEND`、`COUNTER`）
  3. 優先權 3：輔助型（`HEAL`、`MAX_HP_UP`、`STUN`）
  4. 若種類相同，則依能量消耗（SP）升序排序，SP 相同則依卡牌名稱字母排序。
- **好處**：這樣做可以直接讓 `hand` 陣列內部保持有序，從而使 `Battle::drawUI()` 渲染時自然分組，且完全不需要修改 `Battle::handleInput` 中的方向鍵索引移動邏輯（因為視覺順序與索引順序一致）。

### 2. 邊框色彩化渲染
- **決策**：在 `Battle::drawUI()` 中定義一個 Lambda 函式來決定卡牌的渲染顏色。
  - 選取中：`COLOR_LYELLOW`（金黃）
  - 未選取且為攻擊型：`COLOR_LRED`（淡紅）
  - 未選取且為防禦型：`COLOR_LBLUE`（淡藍）
  - 未選取且為輔助型：`COLOR_LGREEN`（淡綠）
- **好處**：提高手牌區域色彩識別度，大幅降低玩家看錯手牌的機率。

### 3. 卡牌高度微增
- **決策**：在 ASCII 卡牌中，新增一行用於顯示種類。這使卡牌整體高度由 5 行變為 6 行。
- **好處**：種類描述可以完美融入卡牌內部，且不影響控制台的整體垂直高度排版。

## Risks / Trade-offs

- **[Risk]** 卡牌高度增加 1 行可能導致低解析度控制台出現捲動。
  - **Mitigation**：程式啟動時已預設將控制台最大化 (`SW_MAXIMIZE`)，螢幕空間非常充裕，1 行的增長不會造成任何顯示異常。
