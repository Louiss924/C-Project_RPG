## Context

目前專案在遊戲啟動後會直接進入第一關 (Level 1-1) 的初始化與對話。為了提升遊戲的完整性與代入感，我們將實作一個經典的控制台主選單畫面，整合「開始冒險」、「遊戲說明」、「離開遊戲」功能。

## Goals / Non-Goals

**Goals:**
- 在 `Game` 類別中新增主選單生命週期與畫面繪製邏輯。
- 實現基於 `_getch()` 捕捉方向鍵 (↑/↓) 與 Enter 鍵的純鍵盤互動選單。
- 新增靜態的「遊戲說明」頁面，顯示卡牌、規則及操作對照，供玩家查閱。
- 確保退出功能可以乾淨地關閉程式。

**Non-Goals:**
- 不改變任何與戰鬥、牌組、關卡數值相關的後台引擎邏輯。
- 不調整現有 Round 2 與 Round 3 之間的 Draft 卡牌選取流程。

## Decisions

### 1. 調整 `Game` 生命週期與控制流程
在 `Game::run()` 的最開始，引入一個主選單的無限迴圈。
- 當玩家選擇「開始冒險」：結束主選單迴圈，進入原本的關卡循環。
- 當玩家選擇「遊戲說明」：呼叫獨立的 `Game::displayInstructions()`，渲染說明文字後，等待按鍵並回歸主選單。
- 當玩家選擇「離開遊戲」：直接呼叫 `exit(0)` 結束程式。

### 2. 主選單選取狀態管理
在 `Game` 內維護一個 `int menuCursor` 成員變數，範圍為 0 到 2：
- 0: 開始冒險
- 1: 遊戲說明
- 2: 離開遊戲
使用 `_getch()` 讀取方向鍵 (`72` 為上，`80` 為下) 遞增或遞減 `menuCursor`，並用 `Enter` (`13`/`10`) 進行觸發動作。

### 3. 主畫面 ASCII Art 設計
採用一個經典的 RPG 標題設計，在控制台中用字元拼出醒目的遊戲名稱，如：
```text
===================================================
      🛡️ 傳奇勇者：卡牌冒險 (Legend of Card Hero) 🛡️
===================================================
```
並在此之下繪製邊框與當前高亮的選單選項。

## Risks / Trade-offs

- **鍵盤緩衝區搶按問題**：玩家如果快速敲擊 Enter 鍵，可能在進入關卡 1-1 後直接跳過對話或誤觸操作。
  - *Mitigation*: 在主選單確認「開始冒險」並跳出迴圈後，先清除一次鍵盤輸入快取：
    ```cpp
    while (_kbhit()) { _getch(); }
    ```
