## Why

本專案旨在從頭建立一個乾淨、結構化的單人卡牌建構（Deckbuilding）C++ RPG 遊戲框架。它為終端機下的回合制卡牌戰鬥、手牌鍵盤操作切換、SP/生命值資源管理以及平滑血量條扣減動畫提供完整的程式結構設計，並確保專案在包含學校電腦在內的 Windows 環境中可順利移轉與無亂碼編譯。

## What Changes

- **專案結構與文件**：
  - 於專案根目錄建立 `README.md`，說明專案目標與學校電腦環境（Dev-C++ / VS Code / MinGW）移轉與編譯指南。
  - 嚴格採用多檔案分離架構：標頭檔置於 `/include`，實作檔置於 `/src`。
  - 入口點為 `/src/main.cpp`，並強制呼叫 `system("chcp 65001 > nul")` 解決 Windows 控制台中文與圖形字元亂碼。
- **卡牌戰鬥系統核心**：
  - 玩家每回合可選擇普通攻擊（不耗 SP，回復 1 SP）或打出卡牌（消耗 SP）。
  - 卡牌包含多種功能：造成攻擊、回復血量、防禦、反擊部分傷害、提高總生命值上限、電暈（怪物暫停一回合）。卡牌抽取為隨機機制。
  - 設計三關（每一關有 3 回合，最後一回合為 Boss 戰）。
  - 意圖預告機制：在玩家出牌前，於畫面上方明確顯示魔王/怪物的下一回合招式名稱與預估傷害。
- **鍵盤控制與文字 UI 動畫**：
  - 全面禁用數字鍵出牌，改用 `<conio.h>` 的 `_getch()` 監聽鍵盤方向鍵（↑/↓）在手牌間切換，並按 Enter 確認出牌。
  - 動畫演出：出牌時鎖定畫面並顯示當前出牌動作。攻擊扣血時，使用 `std::this_thread::sleep_for` 進行血條逐點減少的動態重繪。

## Capabilities

### New Capabilities
- `game-flow-and-stages`: 處理關卡流程（3 個關卡，每關 3 回合，第 3 回合為 Boss 戰）、關卡切換、勝敗判定與遊戲初始化。
- `battle-and-card-core`: 處理回合制戰鬥邏輯、抽牌與手牌管理、SP 能量計算、卡牌效果（傷害、回復、護盾、反擊、最大生命提升、眩暈）與普通攻擊（SP 回復）、以及怪物的行動意圖預告。
- `ui-and-keyboard-control`: 提供基於控制台的 UI 渲染（支援 chcp 65001）、利用 `_getch()` 監聽上下方向鍵切換卡牌與 Enter 鍵出牌、出牌動作鎖定、以及血條漸進式扣減動畫。

## Impact

  - 建立 `README.md`
  - 建立 `/include/` 目錄
