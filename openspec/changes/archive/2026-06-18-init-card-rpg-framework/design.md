## Context

本專案旨在為單人卡牌建構（Deckbuilding）C++ RPG 遊戲建立一個乾淨、模組化的 C++ 專案框架。由於目標運行環境包含學校電腦（通常為 Windows 作業系統，使用 MinGW 或者是 Dev-C++ 編輯器），開發架構必須兼顧「無外部龐大依賴（如 SFML/ncurses）」與「優秀的文字介面動態體驗」。

我們將利用 Windows 內建的 `<conio.h>` 標頭檔讀取鍵盤輸入，並使用標準 `<thread>` 與 `<chrono>` 實作動畫延遲，以及呼叫 `system("chcp 65001")` 確保 UTF-8 中文編碼能正確在終端機顯示。

## Goals / Non-Goals

**Goals:**
- **架構分離**：標頭檔於 `/include`，實作檔於 `/src`，主入口於 `/src/main.cpp`。
- **卡牌與戰鬥引擎**：完整實作卡牌庫、隨機抽牌、手牌更新、能量（SP）消耗與恢復、關卡回合推進、怪物行動意圖預告。
- **互動與動畫**：利用 `_getch()` 捕捉方向鍵以切換手牌，回車鍵出牌；使用毫秒級延遲重繪實現血條扣減動畫。
- **可移植性**：隨附詳細的環境移轉與編譯指南，方便在不同 Windows PC 上直接編譯執行。

**Non-Goals:**
- **跨平台支援**：不考慮 Linux/macOS 的原生支援（因 `<conio.h>` 與 Windows 控制台環境是本設計的核心相依）。
- **複雜卡牌資料庫/存檔**：目前僅提供內置的基礎卡牌種類與 3 大關卡數值框架，不實作外部 JSON/DB 卡牌載入或遊戲存檔功能。

## Decisions

### 1. 模組化類別架構 (Class Architecture)
為了實現多檔案分離，我們將系統劃分為以下五個核心類別：
- `Card` (卡牌類別)：
  - 屬性：名稱、SP 消耗、效果類型（傷害、回復、護盾、反擊、最大生命值提升、電暈）、數值、描述。
- `Player` (玩家類別)：
  - 屬性：當前 HP、最大 HP、當前 SP、護盾值（Armor）、是否處於反擊狀態、反擊率、牌組（Deck）、手牌（Hand）、棄牌堆（Discard）。
  - 方法：抽牌、洗牌、普通攻擊（回復 SP 1）、使用卡牌、受到傷害（優先扣護盾）、重置護盾。
- `Monster` (怪物與 Boss 類別)：
  - 屬性：名稱、當前 HP、最大 HP、是否被電暈（Stunned）、下一回合預告招式與預估傷害。
  - 方法：隨機決定下回合招式、執行招式、受到傷害。
- `Battle` (戰鬥管理器)：
  - 職責：管理單場戰鬥（玩家回合與怪物回合交替）。包含渲染 UI（意圖預告、雙方狀態、手牌列表）、`_getch()` 鍵盤輸入事件循環、出牌動畫鎖定與血條動畫扣減。
- `Game` (遊戲主控器)：
  - 職責：管理大關卡（Level 1-3）與回合（Round 1-3）流程，初始化玩家，產生各關卡怪物，判定勝利/失敗。

### 2. 鍵盤非阻塞/無數字鍵控制方案 (`_getch()`)
- **決策**：棄用傳統 `std::cin` 讀取數字出牌，改用 `<conio.h>` 中的 `_getch()`。
- **機理**：方向鍵（↑/↓）在 Windows 控制台會觸發雙位元組（或逸出序列）輸入。當 `_getch()` 讀到 `0` 或 `224` 時，必須再次呼叫 `_getch()` 讀取實際鍵值（`72` 為上方向鍵，`80` 為下方向鍵）。Enter 鍵對應鍵值為 `13`（或 `10`）。
- **優勢**：操作流暢，防止輸入非數字字元導致 `cin` 崩潰，介面更具遊戲感。
- **替代方案**：使用 `ncurses`。但 Windows 下 `ncurses` 配置繁瑣，極不利於學校電腦環境的移轉與快速編譯，故否決。

### 3. 血條逐點減少與重繪動畫 (HP Decay Animation)
- **決策**：在 `Player` 或 `Monster` 的 `takeDamage` 階段，不直接修改數值並印出。而是以一個迴圈，每次扣除 1 點生命值，隨即呼叫 `Battle::drawUI()` 重新繪製螢幕，並暫停 `std::this_thread::sleep_for(std::chrono::milliseconds(80))`。
- **優勢**：呈現視覺上血條慢慢變短的動畫效果，大幅增強打擊回饋感。

### 4. 戰鬥畫面排版 (Visual Layout)
終端機畫面每次更新時均清屏（使用 `system("cls")` 或 ANSI `\033[H\033[J`），並由上至下印出：
1. **怪物意圖預告區**（如：`【怪物意圖】魔王下回合將使用 [毀滅重擊]，預估傷害: 15`）
2. **戰鬥主畫面**（怪物名稱、血條與數值；玩家名稱、血條、護盾、SP 狀態）
3. **戰鬥日誌區**（鎖定顯示上一次出牌或怪物的攻擊動作說明，如：`▶ 玩家打出 [電暈卡]，使怪物陷入麻痺！`）
4. **手牌與動作選擇區**（包含「普通攻擊」與目前抽到的手牌列表。使用 `>` 箭頭高亮目前游標選中的項目）

## Risks / Trade-offs

- **[Risk] 控制台清屏造成的閃爍 (Flicker)**
  - *Mitigation*: 盡可能減少不必要的重繪。僅在玩家按下方向鍵切換游標、出牌動作確認、以及血條動畫扣減時進行重繪，並優化輸出的緩衝區。
- **[Risk] 學校電腦 Dev-C++ 舊版編譯器相依問題**
  - *Mitigation*: 程式碼完全遵循 C++11 標準編寫，避免使用 C++17/20 新特性。在專案根目錄 README.md 附上 Dev-C++ 與 MinGW 的 gcc 編譯指令說明。
